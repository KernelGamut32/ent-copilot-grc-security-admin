
# Lab 2.4 — Review Oversharing Risk in SharePoint (SAM Governance Scenario)

## Course

Enterprise Copilot Governance, Security, and Administration

## Lab Duration

~60 minutes

## Lab Objective

In this lab you will perform a **SharePoint data access governance review** similar to the reports generated by **SharePoint Advanced Management (SAM)**.

You will:

1. Identify SharePoint sites that grant access to **Everyone except external users**
2. Review site permissions and sharing configuration
3. Identify **potential oversharing risks**
4. Document **remediation recommendations**

> NOTE: Full **SharePoint Advanced Management (SAM)** reporting features require an additional Microsoft license that is not typically included in Microsoft 365 Developer tenants.  
> This lab reproduces the **same governance analysis process** using built‑in SharePoint admin tools that are available in developer tenants.

This exercise demonstrates how organizations detect **overshared content that could be surfaced by Copilot**.

---

## Lab Scenario

Your organization is preparing to deploy **Microsoft 365 Copilot**.

Security leadership is concerned about **overshared SharePoint content**. If content is broadly accessible, Copilot may surface that information to users who should not see it.

The governance team must identify SharePoint sites that grant access to: **Everyone except external users**

These sites are considered **high risk for oversharing**.

Your task is to identify overshared sites and recommend remediation actions.

## Concepts Covered

### Oversharing Risk

Oversharing occurs when content is accessible to more users than intended.

Common risk indicators include:

- Sites shared with **Everyone except external users**
- Large membership groups
- Broad "Member" or "Visitor" access

## Copilot Data Access Model

Microsoft 365 Copilot respects:

- Microsoft Graph permissions
- SharePoint access controls
- Sensitivity labels

If a site is accessible to a user, **Copilot can potentially reference that data**.

Therefore identifying overshared sites is a critical governance task.

<https://admin.microsoft.com/sharepoint>

---

### Task 2 — Review Active Sites

1. In the SharePoint Admin Center select: **Sites → Active sites**

2. Review the list of sites in the tenant.

Developer tenants typically contain sites such as:

- Team sites
- Communication sites
- SharePoint starter content sites

---

### Task 3 — Identify Sites with Broad Access

Select a site from the list and click the site name.

Open: **Permissions → Site permissions**

Look for groups that contain: **Everyone except external users**

If the group appears in:

- Members
- Visitors
- Owners

the site may be overshared.

---

### Task 4 — Document Overshared Sites

Repeat the review process for several sites.

Identify the **top 3 sites with the broadest access**.

Example indicators:

| Indicator | Risk Level |
| --- | --- |
| Everyone except external users in Members group | High |
| Large membership group | Medium |
| Anonymous sharing enabled | High |

---

### Task 5 — Inspect Site Sharing Settings

In the site settings panel review: **Policies → External sharing**

Document whether the site allows:

| Setting | Description |
| --- | --- |
| Anyone | Anonymous access |
| New and existing guests | Guest sharing |
| Existing guests | Restricted sharing |
| Only people in your organization | Internal only |

Sites allowing anonymous or broad internal sharing may increase Copilot exposure risk.

---

### Task 6 — Review Document Library Permissions

Open the site.

Navigate to: **Documents → Settings → Library permissions**

Check whether the library:

- Inherits permissions from the site
- Has unique permissions
- Is shared with broad groups
